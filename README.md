# Кредитный скоринг МФО: предсказание дефолта и сегментация клиентов

Скоринговая модель для микрофинансовой организации: по анкете заёмщика оценивает вероятность дефолта и распределяет клиентов по риск-сегментам A/B/C для принятия кредитного решения. Проект доведён от разведочного анализа до экономической оценки политики выдачи.

> Данные - датасет МФО. Признаки реалистичные и интерпретируемые, что позволяет полноценный feature engineering и интерпретацию модели.

---

## Постановка задачи

**Бизнес-задача.** МФО несёт прямые убытки от невозвратов. Нужно по данным заявки оценить риск дефолта и решить, выдавать ли займ, минимизируя потери, но не отказывая чрезмерно хорошим клиентам.

**ML-задача.** Бинарная классификация (дефолт / не дефолт) на табличных данных с умеренным дисбалансом классов (≈29% дефолтов), с последующим переводом вероятностей в кредитное решение через сегментацию.

## Подход

**1. Разведочный анализ (EDA).** Гипотезо-ориентированный: для каждого признака формулируется гипотеза о связи с дефолтом и проверяется на данных с явным выводом «подтверждена / опровергнута». Обработка пропусков - осмысленная под каждый признак (категориальные -> «Не указано», возраст -> медиана по типу занятости и т.д.).

**2. Препроцессинг.** Вся логика вынесена в класс `CreditPreprocessor` (`BaseEstimator` / `TransformerMixin`). Выученные параметры (медианы) считаются только на train в `fit` и применяются в `transform`. Препроцессинг встроен в `Pipeline`, поэтому при кросс-валидации каждый фолд переучивает статистики на своём train-куске — утечка исключена по построению.

**3. Сравнение моделей.** Логистическая регрессия (бейзлайн) против LightGBM, обе затюнены через Optuna по PR-AUC. Единая метрика и единый отложенный тест для честного сравнения.

**4. Интерпретация (SHAP).** Анализ драйверов риска и сопоставление с выводами EDA.

**5. Сегментация и экономика.** Перевод вероятностей в сегменты A/B/C (пороги — на train) и оценка бизнес-эффекта политики выдачи.

---

## Метрика

Основная метрика - **PR-AUC**. При дисбалансе классов accuracy обманчива: модель, всегда предсказывающая «не дефолт», даёт ≈71% accuracy и нулевую пользу. Дополнительно отслеживается **recall по дефолту** - для МФО пропущенный дефолт (прямой убыток) дороже зря отклонённого клиента (упущенная выручка).

## Результаты

### Сравнение моделей (отложенный тест)

| Модель | PR-AUC | ROC-AUC | Recall (default) | Precision (default) |
|---|---|---|---|---|
| LogReg (baseline) | 0.621 | 0.781 | 0.68 | 0.52 |
| **LogReg (tuned)** | **0.623** | 0.780 | 0.68 | 0.51 |
| LightGBM (tuned) | 0.607 | 0.770 | — | — |

**Финальная модель - логистическая регрессия.** Она превзошла градиентный бустинг и является более простым и интерпретируемым решением.

### Драйверы риска (SHAP)

Главные предикторы дефолта: плохая кредитная история (доминирующий фактор), низкий доход, число прошлых займов, возраст. Направление влияния логично и **совпадает с выводами EDA** - модель уловила реальные закономерности, а не артефакты.

### Сегментация и бизнес-эффект

Клиенты разбиты на сегменты по вероятности дефолта (пороги на train: 0.298 / 0.552). Default rate монотонно растёт от A к C, что подтверждает корректность ранжирования риска.

Политика **«одобрять A и B, отказывать C»**:

| Показатель | Значение |
|---|---|
| Предотвращено дефолтов | 61% от всех |
| Default rate портфеля | 29.4% → 17.0% (−12.4 п.п.) |
| Отклонено заявок | 33% (из них 46% — упущенные хорошие клиенты) |

## Ключевые выводы

**Систематическая проверка гипотез улучшения.** Проверены три пути поднять качество, все дали отрицательный результат — и это задокументировано честно:
1. Градиентный бустинг (LightGBM) - не превзошёл линейную модель.
2. Тюнинг гиперпараметров (Optuna) - без прироста над дефолтом.
3. Feature engineering (debt-to-income, daily payment) - без эффекта; SHAP подтвердил низкую значимость этих признаков.

**Вывод о природе данных.** Совокупность результатов указывает на преимущественно линейные зависимости: Optuna для LightGBM сошлась к мелким деревьям с сильной регуляризацией, а логрег уже извлёк весь предсказуемый сигнал. PR-AUC ≈0.62 (при базовой доле дефолтов 0.29 - вдвое выше случайного) близок к потолку предсказуемости на доступных признаках.

**Выбор модели.** При равном/лучшем качестве выбрана логистическая регрессия - простая, быстрая и интерпретируемая, что критично для кредитного скоринга, где требуется обосновывать отказы.

## Ограничения

- Данные синтетические; реальные зависимости могут быть сложнее.
- Случайный train/test split. В продакшене для скоринга нужен **out-of-time** split (обучение на прошлых заявках, валидация на новых), т.к. модель предсказывает будущее.
- Сегментация оптимизирует долю дефолтов, а не прибыль. Реальная политика должна учитывать LTV клиента, стоимость привлечения и срок отношений - т.е. оптимизироваться по ожидаемой прибыли.
- Для прода потребовались бы вынос признаков в feature store (train/serving consistency) и мониторинг дрейфа данных.
